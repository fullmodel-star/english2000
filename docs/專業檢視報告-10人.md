## 2.【跨專業共識風險】（多專業同時點到，最該優先）

**A. viewport 禁止縮放（`maximum-scale=1, user-scalable=no`，第5行）— 最高共識**
6 個專業同時點名：UX(中)、a11y專家(高)、教育心理學家(中)、前端工程師(低)、特教老師(高)、教材編輯(低)。
這是 WCAG 1.4.4 明文失敗項，弱視／讀寫障礙學生無法雙指放大自救。**一行可解**，零風險，CP 值最高。

**B. 色彩對比大面積不達 WCAG AA（`--muted #8a93a2` 對白僅約 3.1:1、白字按鈕對 `--accent #ff6b6b` 僅 2.78:1，彩底白字計數最低僅 2.03）**
5 個專業點名：UX(高)、a11y專家(高)、教育心理學家(中)、特教老師(中)、教材編輯(低)。
影響「所有學生」長時間小字閱讀，戶外/低階螢幕尤甚。需重算亮/暗兩套 token。

**C. 拼字題零容錯（`spellMatch` 僅容單複數/括號變體，一個字母錯即全錯；現成 `editDist` 只拿來事後分類沒接到批改）**
2 個專業點名且都判高/中：英語老師(中)、特教老師(高)。
讀寫障礙學生最常見漏字母/b-d 顛倒會被直接判錯，第一次挫敗就放棄。改動小、救回最多學生。

**D. 單一詞義/詞性 → 「卡片標示 vs 例句用法」自相矛盾**
2 個專業點名且都判高：英語老師(高)、教材編輯(高)。
實例：watch 標 [n] 手錶但例句 "We watch TV"（動詞）；present 只給「現在的(adj)」漏「禮物(n)」；fine 標「精巧的」非「好的/罰款」；help/call/plan/fish/deal 名動同形卻只鎖一邊。這是**會直接教錯學生**的正確性問題。

**E. 老師作業 due 欄位 innerHTML 未跳脫 → 儲存型 XSS**
2 個專業點名且都判高：家長(低)、前端工程師(中)、資安顧問(高)。
作業檔本就設計在師生間傳遞，攻擊面真實存在。

**F. 每日提醒不可靠（`setTimeout` 離線排程，關 App 即失效）**
3 個專業點名：家長(低)、補習班主任(中)、（資安顧問另就通知內容隱私點名）。
家長/老師誤以為孩子有被提醒，實際沒有 — 是信任落差來源，宣傳時不應當賣點。

## 3.【各專業獨到觀點】（只有該行才會抓到）

- **英語老師：** 「精熟」只看單次測驗/盒子層級，**缺延遲再測把關**——學生當下衝 80% 過關，兩週後早忘，⭐ 與精熟數給家長/老師偏樂觀的假象。
- **家長：** **成就壓力＋失去厭惡話術**（「別讓連續天數斷掉！」）對焦慮/完美主義孩子是壓力源；且**完全沒有家長端可見性/防呆**——自由模式開關就在設定頁，孩子可略過闖關直接全解、一鍵清空紀錄。
- **UX 設計師：** **核心功能「錯題本」未進主導覽**，只能從首頁一顆按鈕進入，發現性低——等於告訴學生它是次要功能。
- **無障礙專家：** **全檔 0 個 aria-* / role / aria-live**，答對答錯/分數/番茄鐘倒數/XP 升級都沒有 live region，螢幕報讀者完全收不到回饋；emoji-only 圖示按鈕（⚙️🔊★✕）無無障礙名稱。
- **教育心理學家：** **XP 與學習品質脫鉤且通膨**——combo 用 `addXp(10+min(combo,6)*2)` 讓「連對速度」而非「學會」主導獎勵，是過度辯證效應(over-justification)的教科書案例；翻卡自評「簡單」直跳 2 個 box 由學生主觀判定、無提取驗證，會把沒學牢的字提早推到長間隔消失。
- **前端工程師：** **`save()` 完全沒有 try/catch 與配額處理**；`history` 每天新增、永不清理，QuotaExceededError 會在熱路徑靜默拋出，使用者無感卻丟資料——唯一在正常使用下會「靜默吃掉學習紀錄」的風險。
- **補習班主任：** **派發-繳交-批改迴路斷掉**——老師派了作業完全收不到成果（無回傳、無學生身分），加上**例句源自出版社教師用書卻無授權聲明**的著作權風險，連鎖補習班商用恐侵權。
- **特教老師：** **「簡潔模式」其實沒真的減量**——只把慶祝彈窗改 toast，沒關 combo/音效/震動；且關「音效」會連帶關掉 TTS 發音，降噪與聽覺輔助兩需求互相衝突；無 `prefers-reduced-motion` 處理。
- **教材編輯：** **例句句框高度重複、模板感重**——"every day" 結尾 27 句、school 出現 114 句，語境過度集中校園日常，不利遷移到會考多情境閱讀；主題分類冷熱極不均（thing=325 大雜燴 vs tech 僅 4）且有錯置（boring 歸 emotion）。
- **資安顧問：** **備份匯入深合併不驗證型別**——外來 JSON 可污染全域狀態 S（塞超大陣列造成本機 DoS、畸形 srs 造成後續例外）；**缺 CSP**，未來任一處忘記跳脫即成 XSS。

## 4.【確認的 Bug / 技術風險】（工程＋資安）

| 風險 | 嚴重度 | 說明 |
|---|---|---|
| `save()` 無 try/catch + 配額處理；`history` 無限成長 | **高** | QuotaExceededError 在 `rate()`/`answerOne()` 熱路徑靜默拋出，記憶體與已存狀態不一致，靜默丟資料 |
| 老師作業 `due` 欄位 innerHTML 未跳脫 → 儲存型 XSS | **高** | `renderBanners` 以樣板字串插入 innerHTML；惡意 `vocab_assignment.json` 含 `<img src=x onerror=...>` 即執行 |
| 拼字輸入經 `addWrong` 存入 `lastPicked`，錯題本/解析渲染未轉義 → 持久型 self-XSS | **中** | 約 25 處 innerHTML 寫入、全檔僅 3 次 escapeHTML |
| 備份匯入 `importData` 整包 Object.assign 不驗證型別/結構 | **中** | 可污染 srs/history/assignment/badges，超大陣列造成本機卡死 |
| 缺 CSP / 權限政策 meta | **中** | 無瀏覽器層縱深防禦；建議 `connect-src 'none'` 把「零外連」從慣例升級為強制 |
| `history` 全量 `Object.values` 加總、每次 save 序列化整包 | **中** | O(n) 寫入放大，成長曲線不健康；建議增量維護 `S.stats` + 7~30 天保留 + debounce |
| `restoreLast()` 存 word 物件快照而非字串陣列 | **低** | WORDS 更新後舊存檔顯示過期資料；無 shape 驗證，損毀存檔讀到 undefined |

**正面：** 零外連、零追蹤、資料全本機；Service Worker 寫得保守正確（同源白名單、版本清舊快取）；狀態層有 v1→dirs 資料遷移，向後相容佳。

## 5.【內容品質】（教材編輯＋英語老師）

**做得好：**
- 例句難度控制專業：2000 句長度 4-11 字（平均 7.3），句型單純皆國中可解碼，幾乎字字有例句、全庫僅 1 句重複、看得出有人工校過。
- 中文釋義/翻譯自然無中式回譯腔（raise→調漲、borrow/lend 借入借出分得清）。
- 測驗誘答設計專業：抓「同詞性＋同主題」當誘答、主動排除中文同義字，鑑別度高（多數自製 App 偷懶處它做對了）。
- 結構完整：2000 字去重無重複、嚴整分 40 單元、Level 1/2 分層、含主題篩選+精熟度面板。

**主要扣分（正確性風險，比例句重複更傷教學信任）：**
- **單一詞性標註造成卡片 vs 例句矛盾**（最高優先）：watch/help/call/plan/fish/deal/respect 等名動同形高頻字標 [n] 卻用動詞例句，會建立錯誤詞性概念，測驗考詞性將直接出錯。
- **預設詞義不是會考最常考/最先學的義項**：present 漏「禮物」、fine 非「好的/罰款」、little 非「小的」、fall 非「落下/秋天」——學生閱讀遇另一義項會卡住甚至以為背錯。建議先人工補一份 80~120 字高頻多義字清單。
- 例句語境過度集中校園日常，模板感重，不利遷移。
- 主題分類冷熱極不均且有錯置。
- 選字排除 be/人稱代名詞等功能詞但未說明依據，缺英文 `lang="en"` 標記。

## 6.【優先處理清單】（排序）

1. **移除 viewport `maximum-scale=1, user-scalable=no`**（無障礙紅線，一行可解，零風險）— 🟢**跨專業共識（6 個專業）**
2. **修老師作業 `due` XSS**：渲染端 `escapeHTML(due)` + 匯入端白名單 `/^\d{4}-\d{2}-\d{2}$/`，並一併跳脫 note 與所有字串渲染點 — 🟢**跨專業共識（資安/工程/家長）**
3. **`save()` 加 try/catch + 配額處理 + `history` 保留期裁剪**（唯一會在正常使用下靜默丟學習紀錄的風險）— 🟡**單一專業（前端工程師，高）**
4. **補高頻多義字第二義項 / 修正詞性矛盾**（盤點 80~120 字：present/fine/little/fall/watch/help/call/plan…，補進資料並顯示於卡背與測驗解析）— 🟢**跨專業共識（英語老師/教材編輯，皆高）**
5. **拼字題加 `editDist≤1`「差一點」容錯 + 重試一次**（接現成 editDist 到批改流程，可設「寬鬆拼字」開關預設給資源班）— 🟢**跨專業共識（英語老師/特教，特教判高）**
6. **全面修色彩對比與字級**：`--muted` 加深至 ≥4.5:1、正文 14px 起跳、按鈕白字與成功綠達標、亮/暗兩套 token 都驗 — 🟢**跨專業共識（5 個專業）**
7. **減壓/家長友善模式**：可隱藏 streak 與會考倒數、文案改正向鼓勵、streak 容許每週補做、加每日螢幕時間上限與護眼休息提示，獎勵改綁「精熟事件」而非 combo — 🟢**跨專業共識（家長/教育心理學家）**
8. **補班級閉環**（若要入校）：學生端一鍵匯出含座號姓名的精簡進度 JSON + 一張離線老師彙整頁；強制填寫班級/座號/姓名 profile；釐清例句著作權與授權聲明 — 🟡**單一專業（補習班主任，高），但為商業導入唯一門檻**

**補充快贏（低成本可一併做）：** 加 CSP meta（`connect-src 'none'`）；`prefers-reduced-motion` 全域處理；emoji 圖示按鈕補 aria-label；錯題本提進主導覽或首頁置頂「今日待複習 N 題」卡。
