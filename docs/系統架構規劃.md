# 國中英語複習系統 — 開發系統架構（v1 待確認）

> 目標：單機、離線、多帳號的國中英語複習 PWA。學生選單元、由易到難、間隔複習；
> 老師可在本機看各身分統計。題庫以 Excel 維護、一鍵重建。無伺服器、零成本。

---

## 1. 整體架構（兩層）

```
╔══════════ 維護層（老師/開發端，平時不給學生碰）══════════╗
║                                                          ║
║   Excel 題庫  ⇄  master JSON（正規來源）                  ║
║   (人工編輯)        │                                     ║
║                     │ build_app.py（驗證 + 注入）         ║
║                     ▼                                     ║
║              index.html（內嵌題庫資料）                   ║
╚══════════════════════════╤═══════════════════════════════╝
                           │ 發佈「單一 HTML 檔」
                           ▼
╔══════════ 執行層（學生裝置 / 瀏覽器，離線）═══════════════╗
║  index.html (PWA, 可加到主畫面)                          ║
║  ┌───────────────────────────────────────────────────┐  ║
║  │ 題庫資料（唯讀，內嵌）  單字 / 文法 / 閱讀          │  ║
║  ├───────────────────────────────────────────────────┤  ║
║  │ 身分模組   profiles + 目前身分 (localStorage)       │  ║
║  │ 進度模組   每身分各自 SRS / 錯題 / 答題記錄         │  ║
║  │ 複習引擎   選單元 → 由易到難 → 間隔複習             │  ║
║  │ 統計模組   個人統計 + 老師總覽                      │  ║
║  │ 匯出匯入   進度 JSON 檔（換裝置 / 老師收集）        │  ║
║  └───────────────────────────────────────────────────┘  ║
╚══════════════════════════════════════════════════════════╝
```

**技術選型**：純 HTML/CSS/JS 單檔（沿用現有 index.html 風格，無框架相依）＋ PWA(manifest + service worker 離線)。題庫於建置時內嵌成 JS 常數，發佈為單一檔。

## 2. 資料層

### 2.1 統一題目模型（三模組共用）
```jsonc
{
  "id": "G-tense-0001",
  "type": "grammar",          // grammar | reading | vocab
  "unitGroup": "動詞類",       // 選單大分組
  "unit": "時態",              // ★複習單元（學生選這個）
  "difficulty": 2,            // ★1–3 由易到難（逐題計算）
  "level": 1,                 // 教材原始分級（參考用）
  "genre": null,              // 閱讀文體；文法/單字為 null
  "stem": "When John ___ his teeth, the phone rang.",
  "options": ["brushes","brushed","was brushing","has brushed"],
  "answer": 2,                // 正解索引 0–3
  "explanation": "……",
  "explSource": "ai",         // ai | origin（解析來源，透明標示）
  "passageId": null,          // 閱讀題指向文章；其餘 null
  "source": "文法特快車",
  "flags": { "ocr": false, "answerCheck": "verified" }  // verified | review
}
```

### 2.2 閱讀文章（與題目分開存）
```jsonc
{ "passageId":"R-0007", "genre":"書信", "level":2,
  "hasText":true, "text":"Dear Mr. President, ……" }
```

### 2.3 單元分類（定案）
- **文法 → 16 統一主題**（合併特快車＋精熟，拆開混合範疇）
  - 動詞類：時態 / 被動語態 / 動名詞與不定詞 / 使役感官特殊動詞 / 助動詞
  - 名代形副：名詞與冠詞 / 代名詞 / 形容詞 / 副詞 / 比較級
  - 介連：介系詞 / 連接詞與對等子句
  - 子句：名詞子句 / 形容詞子句 / 副詞子句
  - 句型：假設語氣 / 附加問句·倒裝·感嘆 / there be 句型
- **閱讀 → 雙維度**：程度(L1/L2/L3) × 文體(書信/海報/對話/圖表/記敘/說明…)
- **單字 → 主題 + 分級**（基礎1200 / 挑戰800，沿用既有）

### 2.4 難度計算（逐題，1–3）
綜合客觀指標加權：
1. **字彙難度（主訊號）**：題中單字落在 基礎1200 / 挑戰800 / 超綱 的比例
2. 題幹長度、子句數
3. 題型（單格 < 兩格 < 句意理解 < 克漏字）
4. 範疇本身難度權重（可調表）
→ 算出分數後分箱為 1/2/3。**權重以表格管理、可微調、可重跑**。

## 3. 儲存層（localStorage，本機多帳號）

```
app.profiles   = [{ id, name, emoji, createdAt }]
app.currentId  = "p_ab12"
prog.<pid>     = {
   srs:   { "<qid>": { ease, interval, dueDay, reps, lapses, lastDay } },
   wrong: [ "<qid>", … ],                 // 錯題本
   log:   [ { qid, correct, chosen, ts } ],// 答題記錄
   stats: { byUnit:{}, byType:{}, streak, total, correctRate }
}
```
- 每身分資料**獨立命名空間**（`prog.<pid>`），切換身分＝切資料。
- 題庫本身唯讀、不進 localStorage（內嵌於程式）。

---

## 4. 複習引擎（間隔複習 SM-2 精簡版）

```
選類型(文法/閱讀/單字) → 選單元(如「假設語氣」)
   → 取該單元題目，未學過的「依 difficulty 由易到難」逐步introduce
   → 已學過的依 SRS dueDay 到期者優先
   → 作答 → 即時對錯 + 顯示解析 → 更新該題排程
         答對：間隔拉長；答錯：進錯題本、間隔重置
```
- 每身分、每題各自記 ease/interval/due。
- 「由易到難」體現在**新題引入順序**＝difficulty 升冪。

## 5. 畫面 / 功能模組

| # | 畫面 | 功能 |
|---|------|------|
| 1 | 身分選擇 | 點名字進入；新增/改名/刪除身分（免密碼）|
| 2 | 首頁 | 今日待複習數、選類型、最近統計 |
| 3 | 單元選擇 | 文法16主題 / 閱讀程度×文體 / 單字主題 |
| 4 | 作答 | 出題、選答、即時對錯、看解析、進度條 |
| 5 | 錯題本 | 該身分答錯題，可重練 |
| 6 | 統計 | 個人：各單元熟練度、正確率、連續天數 |
| 7 | 老師模式 | 本機所有身分的進度/正確率/錯題總覽 |
| 8 | 設定 | 匯出/匯入進度、清除、關於 |

---

## 6. 匯出 / 匯入（無後端的「收進度」）
- 匯出：把 `prog.<pid>` 串成 JSON 檔（或長字串/QR）。
- 匯入：老師或新裝置讀入 → 還原該身分進度。
- 用途：換裝置接續；老師集中檢視（學生交檔，老師匯入看）。

---

## 7. 建置 / 維護管線
```
master JSON（正規來源，含三模組題庫 + 文章）
   ├─ 匯出 Excel（老師編輯：改答案、加題、調難度/單元）
   ├─ 匯入 Excel → 回寫 JSON（validate：選項數、答案範圍、必填）
   └─ build_app.py：把 JSON 注入 index.html 的 DATA 區 → 產出單一 HTML
```
- **單一真相來源 = master JSON**；Excel 是編輯介面、雙向同步。
- 每次建置自動驗證資料完整性（壞題擋下並報出）。

## 9. 與現況的落差（要做的轉換工作）
| 項目 | 現況 | 目標 |
|------|------|------|
| 文法分類 | 兩套範疇(10+19)、有混合 | 16 統一主題、逐題歸類 |
| 難度 | 系列粗分 1/2(/3) | 逐題 1–3 計算 |
| 閱讀單元 | 只有程度 | 程度 × 文體 |
| 題目格式 | 三檔 schema 不一 | 統一題目模型 |

## 10. 風險 / 限制（誠實標註）
- 純本機：進度不跨裝置自動同步（靠匯出匯入彌補）；老師無法「遠端即時」看進度。
- 免密碼：同裝置上身分可被他人點選（教室情境可接受）。
- OCR/AI 來源題目：答案/解析非 100% 權威，以 flags 標示、需老師抽查。
- 圖片「選項」題（少數）無法純文字化，不納入。

---
